# OpenSkulls
CLI workflow tool for anything and everything related to structoring and makning of skills, rules, instructions, agents etc for any repo.

## Initialize Workflow
1. Analyse the repo, identify coding languages, dialects, codebase, workflows, frameworks, etc. strengthen this claude
2. Identify the llm or ai cli tool available - ask the user in cli menu from drop down.
3. Analyse the workings, functionality, inteactions and usage of the codebase
4. Analyse the intended usages of the codebase 
5. Analuse the intended external interaction of the codebase
6. Build: skills, rules, instructions. etc based on the used cli

## What Goes where
What rules, skills etc. are local for the individual developer, what are specific to the project.
This can very on how the initiator of this program intends the usage of the code, is it to make it easier for others to adopt and agentic engineering approach, is it to based on the individual coding style etc
